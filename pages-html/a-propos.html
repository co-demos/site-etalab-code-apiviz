D'où viennent les données ?

Dans le cadre de la Politique de contribution de l'État aux logiciels libres, la DINSIC collecte la liste des comptes d'organisation où des organismes publics publient leurs codes sources. Nous nous servons de cette liste pour collecter des métadonnées sur tous les dépôts de code source. Ces métadonnées sont publiées sur data.gouv.fr ou requêtables depuis cette API.


La publication des codes sources est imposée par la Loi pour une République numérique : tout code source obtenu ou développé par un organisme remplissant une mission de service public est considéré comme un « document administratif » devant être publié en open data. Pour connaître les conditions d'ouverture d'un logiciel du secteur public, vous pouvez consulter ce guide interactif.


Je ne vois pas mes codes sources !

Écrivez-nous et nous ajouterons votre forge ou votre compte d'organisation.


En quoi cette liste peut m'être utile ?

Votre organisme accomplit une mission de service public ? Avant de développer du code source par vous-même, vous pouvez chercher si du code déjà développé n'est pas disponible. Vous pouvez aussi repérer des projets qui vous intéressent pour vous rapprocher des organismes porteurs et leur demander comment contribuer.


Vous vous y connaissez en code ? Vous pouvez trouver des projets qui vous intéressent et contribuer.

Puis-je partager un lien vers une requête ?

Oui ! Vous pouvez utiliser les paramètres "s", "g", "language" et "license" comme dans ces exemples :


  • Trouver les dépôts contenant "API" : https://code.etalab.gouv.fr/fr/repos?q=API
  • Trouver les dépôts des groupes contenant "beta" : https://code.etalab.gouv.fr/fr/groups?g=beta
  • Trouver les dépôts en Python : https://code.etalab.gouv.fr/fr/repos?language=python
  • Trouver les dépôts sous Affero GPL : https://code.etalab.gouv.fr/fr/repos?license=Affero
  • Combiner les requêtes : https://code.etalab.gouv.fr/fr/repos?license=Affero&g=beta
